# 流量字段枚举

> 对应 `TrafficRecord` Pydantic 模型，共 13 字段，Pandera `TrafficFormatSchema` 声明式校验。

| 类别 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| **请求** | `method` | str | HTTP 方法: GET/POST/PUT/DELETE |
| | `url` | str | 请求 URL（含 path + query） |
| | `query_params` | str | Query 参数字符串 |
| | `referer` | str | Referer 来源页 |
| | `user_agent` | str | User-Agent 浏览器标识 |
| | `cookie_length` | int | Cookie 长度 |
| **网络** | `source_ip` | str | 来源 IP 地址 |
| | `x_forwarded_for` | str | X-Forwarded-For 代理链 |
| **响应** | `status_code` | int | HTTP 状态码 (100-599) |
| | `response_time_ms` | int | 响应耗时（毫秒） |
| | `content_type` | str | 响应 Content-Type |
| **时间** | `request_time` | str | ISO8601 请求时间戳 |
| **标签** | `identity_label` | str | 身份标签: real / fake |

### Pandera 校验规则

**格式校验** (`TrafficFormatSchema`, 13 字段):
- `method` ∈ {GET, POST, PUT, DELETE}
- `url` 以 `http` 开头
- `status_code` ∈ [100, 599]
- `source_ip` 符合 IPv4 正则
- `response_time_ms` ≥ 0
- `cookie_length` ≥ 0

**业务校验** (`TrafficBusinessSchema`, 6 跨字段规则):
- `status_code=200` 时 `response_time_ms < 5000`
- `method=POST` 时 URL 不得为静态资源路径
- 行业相关 URL 模式匹配（如 ecommerce → `/api/order`, `/api/product`）
